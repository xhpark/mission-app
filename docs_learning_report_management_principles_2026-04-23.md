# 학습 및 리포트 관리 원칙 (v1)

작성일: 2026-04-23  
적용 대상: 미션 언어 학습 앱(학습자 앱)

## 1) 목적
- 학습 흐름을 끊지 않으면서도 학습 데이터 정합성을 유지한다.
- 학습자 주도형 리포트 제출 정책을 명확히 한다.
- 이어하기/새로 시작/오프라인 상황에서의 처리 원칙을 통일한다.

## 2) 핵심 원칙
- 원칙 A. 학습 콘텐츠 우선순위  
문장/단어/오디오는 앱 내 로컬 콘텐츠를 기준으로 제공한다.

- 원칙 B. 누적 학습 기록  
학습 기록은 리포트 제출 전까지 누적 보존한다.  
짧게 학습했다고 매번 리포트를 강제하지 않는다.

- 원칙 C. 리포트 제출 시점  
리포트는 학습자 의사에 따라 언제든 제출 가능하다.  
주간 제출(예: 최소 주 1회)은 운영 정책으로 안내하되, 앱에서 과도하게 강제하지 않는다.

- 원칙 D. 이어하기와 새로 시작  
활성 세션이 있으면 이어하기를 우선 안내한다.  
학습자가 명시적으로 새로 시작을 선택하면 기존 활성 세션은 중단(abandoned) 처리 후 새 세션을 시작한다.

- 원칙 E. 오류/오프라인 내구성  
네트워크/서버 오류 시 재시도, 오프라인 큐, 재동기화를 통해 데이터 손실을 방지한다.

## 3) 세션 상태 정책
- active: 현재 진행 중 세션
- pending_report: 학습 종료, 리포트 대기 상태
- submitted: 리포트 제출 완료 상태
- abandoned: 학습자 선택으로 중단된 상태(새로 시작 시 이전 활성 세션)

## 4) UX 가이드 문구 기준
- 학습 시작 전 학습가이드에서 본 문서의 요약 원칙을 노출한다.
- "원칙을 확인했습니다" 체크 후 학습 시작 버튼을 활성화한다.
- 핵심 문구:
  - "학습 기록은 누적 관리됩니다."
  - "리포트는 원하는 시점에 제출할 수 있습니다."
  - "새로 시작 시 기존 활성 세션은 중단 처리됩니다."

## 5) 운영/개발 유의사항
- 운영 빌드에서는 임시/샘플 진입 버튼을 노출하지 않는다.
- 학습/테스트 화면은 활성 세션 기반 진입을 기본으로 한다.
- 로컬 콘텐츠(문장/단어/오디오) 개수와 매핑 정합성을 정기 점검한다.
